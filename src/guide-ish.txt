//Init sequence\\
call Init_Wrapper
continue as usual for the different layers of gde initialization
once init is complete and after mainloop startup is complete you can fetch the class methods
call the method lists you want to init
there will be two versions to test:
- library stored which you'll only need to call, but will need to fetch the struct exported by the lib
- you own the lifetime of the struct which holds this. You will need to pass the pointer to the struct for the lib to populate it.
(If I keep this a shared lib I'm thinking option one is better. If multiple extensions want to use the wrapper you only need one shared lib to store the method pointers.)


/*classes*/
Class inits will be in two flavours.
- with the suffix _Intern you do not need to pass anything. However you do need to use the internal struct in order to get and use the method pointers.
- without the suffix. These you need to pass a struct to in order to initialize. Once initialized you no longer need to return to the library, because you own the memory.




/*Use to initialize the Godot API. Includes builtins, the API itself and variant to/from methods.
* Pass in the InterfaceGetProcAddress
* no return
*/
Init_Wrapper :: proc "c" (p_get_proc_address : GDE.InterfaceGetProcAddress)
